# Review Policy

## Purpose

This document defines mandatory review behavior for all agents.

Every completed task must be reviewed before being considered finished.

---

## Review Objectives

Verify:

- Correctness
- Stability
- Scope compliance
- Architecture compliance
- SEO compliance
- Accessibility compliance

---

## Required Review Areas

### Functional Review

Verify that the requested functionality works as intended.

### Scope Review

Verify that only necessary files were modified.

### Architecture Review

Verify that existing architecture remains intact.

### Design Review

Verify consistency with design system.

### SEO Review

Verify:

- semantic HTML
- metadata compliance
- heading hierarchy

### Accessibility Review

Verify:

- keyboard accessibility
- semantic structure
- alt text usage

---

## Scope Violation Detection

Flag review failure when:

- unrelated files are modified
- unnecessary refactors occur
- architecture changes occur without approval

---

## Review Outcome

Every review must produce:

PASS

or

FAIL

with justification.

---

## Approval Rule

Completion does not imply approval.

Work is considered complete only after successful review.
